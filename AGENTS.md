# AGENTS.md

Жесткие правила работы с репозиторием `CargoTP`. Следуй им при любом изменении кода, данных или документации.

## Главные Правила

- Всегда работай из корня репозитория: `/Users/walterpng/Documents/tp/CargoTP`.
- Основная ветка только `main`.
- После каждого завершенного изменения делай commit и push в `origin/main`, если пользователь явно не попросил паузу.
- Перед началом изменений выполняй `git status -sb`.
- Никогда не перетирай чужие или пользовательские изменения без прямого разрешения.
- Не используй destructive git-команды вроде `git reset --hard` или `git checkout -- <file>` без явной команды пользователя.
- Не коммить секреты, токены, `.env`, временные файлы, локальные логи и случайные артефакты.
- Не добавляй demo, seed, fake или fallback business data. Если данных нет, UI должен явно показывать состояние "нет данных".

## Источник Данных

- Единственный production Excel source of truth: `data/raw/Олексієнко.xlsx`.
- Допустимый архивный формат после обработки: `data/raw/Олексієнко_DD.MM-DD.MM.xlsx`.
- `data/processed/sales.json`, `group-plan.json`, `receivables.json`, `meta.json`, `dashboard.json` считаются generated output.
- Не редактируй generated processed JSON руками, кроме осознанной правки `monthly-plans.json` и синхронизации `dashboard.monthlyPlans`.
- Перед запуском `npm run process:data` помни: скрипт может архивировать/переименовать raw Excel-файл.
- После любой обработки Excel обязательно запускай `npm run audit:data` и `npm run verify:source`.
- Запрещено менять правила исключения шин из валового плана без явного требования пользователя.

## Production UI

- GitHub Pages использует корневой `index.html`.
- Если меняешь пользовательское поведение дашборда для production, сначала обновляй `index.html`.
- Next.js-код в `app/`, `components/`, `lib/` поддерживай согласованным, если изменение должно жить и в расширяемой версии.
- Не добавляй marketing landing page вместо рабочего dashboard-интерфейса.
- UI должен оставаться полезным при пустых данных: понятный статус, без выдуманных KPI.
- Месяцы для выбора строятся только из дат продаж, а не из ручных планов.
- Валовый план считается по обороту без шин.

## Код И Архитектура

- Используй TypeScript и существующие типы из `lib/types.ts`.
- Для аналитики и бизнес-логики сначала ищи существующие helpers в `lib/analytics.ts`, `lib/product-groups.ts`, `lib/format.ts`.
- Для Excel-парсинга расширяй aliases и схемы в `lib/excel.ts`, а не добавляй отдельный парсер рядом.
- Для данных используй структурный JSON/Excel parsing, не хрупкий string parsing.
- Держи изменения маленькими и привязанными к задаче.
- Не добавляй новую зависимость без реальной необходимости.
- Не добавляй lockfile или package-manager metadata случайно; если меняешь зависимости, объясни и проверь последствия.
- Сохраняй текущий стиль проекта: простой TypeScript, понятные функции, минимум абстракций.

## Проверки Перед Commit

- Для docs-only изменений: `git diff --check`.
- Для TypeScript/UI изменений: `npm run typecheck`, если зависимости установлены.
- Для data pipeline: `npm run process:data`, `npm run audit:data`, `npm run verify:source`.
- Для static dashboard UI: проверить `index.html` через static server, если изменение влияет на загрузку или рендер.
- Если проверку нельзя выполнить, зафиксируй причину в финальном сообщении.

## Git Протокол

- Commit message пиши коротко и по смыслу, например `docs: add project worklog and agent rules`.
- Перед commit проверяй `git diff --stat` и `git diff --check`.
- Коммить только файлы, относящиеся к задаче.
- Push всегда в `origin main`.
- После push проверь `git status -sb`, чтобы рабочее дерево было чистым.

## Красные Линии

- Нельзя коммитить токены Telegram или GitHub.
- Нельзя возвращать demo values вроде `Авто Плюс`, `Brand A`, `U-100`, `C-100-A`.
- Нельзя скрывать ошибку данных под красивыми KPI.
- Нельзя менять source of truth с `Олексієнко.xlsx` на другой файл без явного решения пользователя.
- Нельзя чинить static UI только в Next.js и считать задачу завершенной для GitHub Pages.
- Нельзя запускать обработку Excel как "проверку" без понимания, что она меняет файлы.
